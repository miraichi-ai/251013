自動返信ワークフローの完全図解

🎯 全体の流れ：フォームからAI返信までの5ステップ

📌 結論
この仕組みは「フォーム→AI→返信生成→シート反映」という自動データパイプラインです。人の手を介さずに、問い合わせを受けてからAIが返信案を作成し、スプレッドシートに記録するまでが完全自動化されています。
💡 背景
従来は問い合わせを受けたら、人がメール文を作成し、手動でコピー＆ペーストしてデータ整理をしていました。この作業には時間がかかり、ミスも発生しやすい状況でした。そこで、Googleの標準機能とDify APIを組み合わせることで、この一連の流れを自動化することが可能になりました。

1️⃣ ステップ1：Googleフォーム送信

📋 フォームからスプレッドシートへの自動入力
📌 結論
ユーザーがGoogleフォームを送信すると、自動的にスプレッドシートに1行が追加されます。この時点でトリガーが発動し、GASが動き始めます。
💡 背景
Googleフォームは標準機能として、送信されたデータを自動的にスプレッドシートに記録します。この機能を利用することで、データ入力作業を完全に自動化できます。さらに、フォーム送信時に「onFormSubmit」というイベントが発生するため、これをトリガーとして次の処理を開始できます。
📝 データ構造の例
A列 B列 C列 D列 I列
Timestamp 名前 メール 会社名 ID（a000001）
2025/10/13 14:30 山田太郎 yamada@example.com ABC株式会社 a000001

2️⃣ ステップ2：onFormSubmit関数の実行

⚙️ フォーム送信トリガーの自動起動
📌 結論
フォーム送信をトリガーとして、GASのonFormSubmit関数が自動実行され、最新のデータを取得してJSON形式に変換します。
💡 背景
Google Apps Scriptには「トリガー」という機能があり、特定のイベント（フォーム送信、時間、スプレッドシート編集など）をきっかけに自動的に関数を実行できます。この仕組みを使うことで、人が介入することなく、送信と同時にデータ処理が開始されます。 
function onFormSubmit(e) {
  // スプレッドシートの最終行を取得
  const sheet = SpreadsheetApp
    .getActiveSpreadsheet()
    .getSheetByName('Form Responses 1');
  const lastRow = sheet.getLastRow();
  
  // ヘッダーとデータを取得
  const headers = sheet.getRange(1, 1, 1, 9).getValues()[0];
  const data = sheet.getRange(lastRow, 1, 1, 9).getValues()[0];
  
  // JSON化
  const jsonData = {};
  for (let i = 0; i < headers.length; i++) {
    jsonData[headers[i]] = data[i];
  }
  
  // Difyに送信
  sendToDify(jsonData);
}
📝 JSON化の具体例
スプレッドシートの行データ：
["2025/10/13 14:30", "山田太郎", "yamada@example.com", ...]

↓ JSON化後 ↓

{ "Timestamp": "2025/10/13 14:30", "名前": "山田太郎", "メール": "yamada@example.com", ... }

3️⃣ ステップ3：sendToDify関数でAI呼び出し

🚀 Dify APIへのデータ送信
📌 結論
JSONデータをDify Workflow APIに送信し、AIに返信文の生成を依頼します。APIキーとURLはスクリプトプロパティから安全に取得します。
💡 背景
Difyは、ノーコード/ローコードでAIワークフローを構築できるプラットフォームです。Workflow APIを使うことで、外部システム（今回はGAS）からAIの処理を呼び出すことができます。APIキーなどの機密情報は、スクリプトプロパティに保存することでコードに直接書かずに安全に管理できます。 
function sendToDify(data) {
  // APIキーとURLを取得
  const difyUrl = PropertiesService
    .getScriptProperties()
    .getProperty('DIFY_API_URL');
  const difyKey = PropertiesService
    .getScriptProperties()
    .getProperty('DIFY_API_KEY');
  
  // リクエストボディを作成
  const payload = { inputs: data };
  
  // HTTPオプション設定
  const options = {
    method: 'post',
    headers: {
      'Authorization': `Bearer ${difyKey}`,
      'Content-Type': 'application/json'
    },
    payload: JSON.stringify(payload)
  };
  
  // Dify APIにPOST送信
  const response = UrlFetchApp.fetch(difyUrl, options);
  Logger.log(response.getContentText());
}
📝 Dify側での処理
Difyのワークフローでは、入力された情報（名前、問い合わせ内容など）を元に、AIモデル（GPT-4など）が件名と本文を生成します。例えば：
  • 入力：問い合わせ内容「納期について知りたい」
  • AI処理：過去のデータやテンプレートを参照して適切な返信を生成
  • 出力：件名「お問い合わせいただいた納期について」、本文「お問い合わせありがとうございます...」

4️⃣ ステップ4：DifyからのWebhook受信（doPost）

📨 AI生成結果を受け取る
📌 結論
DifyがAI生成を完了すると、Webhook経由でGASのdoPost関数にJSON形式で結果が返されます。この結果をdifyresponseシートに記録します。
💡 背景
Webhookとは、あるシステムから別のシステムにリアルタイムでデータを送る仕組みです。DifyのワークフローでWebhook URLを設定しておくことで、AI処理が完了した瞬間に自動的にGASに結果を送信できます。GASのdoPost関数は、外部からのPOSTリクエストを受け取るための特別な関数です。 
function doPost(e) {
  // POSTされたJSONを解析
  const json = JSON.parse(e.postData.contents);
  
  // difyresponseシートを取得
  const sheet = SpreadsheetApp
    .getActiveSpreadsheet()
    .getSheetByName('difyresponse');
  
  // JSONからデータを抽出
  const id = json.ID;
  const subject = json.subject;
  const body = json.body;
  
  // シートに追加
  sheet.appendRow([id, subject, body, new Date()]);
  
  // レスポンス返却
  return ContentService.createTextOutput('OK');
}
📝 受信データの例
DifyからのJSON：
{"ID":"a000001", "subject":"お問い合わせの件について", "body":"この度はお問い合わせいただき..."}

これがdifyresponseシートに以下のように記録されます：
A列（ID） B列（件名） C列（本文） D列（日時）
a000001 お問い合わせの件について この度はお問い合わせいただき... 2025/10/13 14:31

5️⃣ ステップ5：XLOOKUP関数で自動反映

📊 メインシートへのデータ統合
📌 結論
スプレッドシートのXLOOKUP関数を使って、difyresponseシートのデータをメインシート（Form Responses 1）に自動的に反映させます。
💡 背景
XLOOKUP関数は、指定した値を別のシートから検索して対応するデータを取得する強力な関数です。今回の仕組みでは、各問い合わせに付与されたID（I列）をキーとして、AIが生成した件名と本文を検索し、メインシートに表示させます。これにより、データが複数のシートに分散していても、一元的に確認できます。
📝 数式の設定例
K列（件名案）：
=XLOOKUP(I2, difyresponse!A:A, difyresponse!B:B, "")

L列（本文案）：
=XLOOKUP(I2, difyresponse!A:A, difyresponse!C:C, "")

列 内容 説明
I列 ID 各問い合わせの一意の識別番号
K列 件名案 AIが生成したメール件名
L列 本文案 AIが生成したメール本文

🎯 まとめ：この自動化システムの本質

この仕組みは、単なる「自動返信システム」ではありません。以下の3つの要素が統合された、現代的なデジタルワークフローです：

  • データ駆動型：すべての情報がデジタル化され、追跡可能
  • AI活用：人間の判断が必要な部分をAIが補完
  • ローコード実装：専門的な開発スキル不要で構築可能

💡 ポイント：これは「問い合わせAIワークフローの最小実装版」であり、この基本形を理解すれば、他の業務自動化にも応用できます。例えば、注文受付、予約管理、アンケート分析など、様々な分野に展開可能です。
